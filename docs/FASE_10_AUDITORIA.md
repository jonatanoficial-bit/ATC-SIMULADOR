# Auditoria — Fase 10

A Fase 10 adiciona uma camada de acessibilidade e configurações com validação automática e auditoria em navegador.

## Critérios de aprovação

- O painel precisa abrir pelo menu, pelo botão do turno, pelo HUD desktop/tablet, pelo botão mobile e pelos atalhos F10/Alt+S.
- As preferências precisam alterar `dataset`, classes CSS e variáveis visuais sem quebrar o jogo.
- O reset precisa retornar ao padrão.
- O JSON exportado precisa refletir as configurações atuais.
- Nenhum erro de console ou exceção de página pode ocorrer.
- As quatro resoluções oficiais precisam permanecer utilizáveis.

## Resultado esperado

Todos os testes F10 devem passar junto das regressões acumuladas das fases anteriores.
